Platform Updates

This page contains updates to components included with the ASCOM Platform. Be sure to check here periodically to make sure you have the latest versions of platform components. Developers should also see the Platform Tools Updates and Additional Tools pages.

IMPORTANT: If you do not have your computer set for Windows Automatic Updates, you must keep up with Microsoft .NET Framework updates yourself. As of November 2009, you must update to one of the following:

  • Windows XP, Windows Vista, Windows 7 - Microsoft .NET Framework 3.5 Service Pack 1
  • Windows 2000 only - Microsoft .NET Framework 2.0 Service Pack 2
Platform Updater 5.5.1
RECOMMENDED - Integrated Platform updater - Download the run this installer to update your ASCOM Platform to version 5.5. See the .NET note above! This update contains a few bugfixes, a critical update to the .NET Client Toolkit, and a new set of driver helper components that (among other things) provide much greater reliability with USB to serial ports. This update further extends our support of Vista, Windows 7, and 64-bit systems.
FilterWheel Simulator (5.0.1a)
FilterWheel simulator. This was not included in the Platform 5 distribution. It passes Conform tests. The (optionally installed) sources include the Conform log. 5.0.1a fixes installation in countries where the point character is not the decimal separator.
Plain Old Telescope Handset (POTH) Update (5.0.5)
This update version has better reporting of errors under several circumstances.
ASCOM Camera Simulator Update (5.0.9a)
This version passes all Conformance Tool tests and has a corrected friendly name for object browsers. This updater will remove uninstallation info for previous updates that were packaged as a separate driver installer. 5.0.9a fixes installation in countries where the point character is not the decimal separator.
ASCOM Dome Simulator Update (5.0.8a)
This version passes all Conformance Tool tests and has a corrected friendly name for object browsers. This updater will remove uninstallation info for previous updates that were packaged as a separate driver installer. 5.0.8a fixes installation in countries where the point character is not the decimal separator.
ASCOM Rotator Simulator Update (1.0.3)
This release fixes: a) setup 1.0.2 did not register the simulator in the ASCOM Profile store and b) the simulator was not visible in the Windows 7 task bar when running.
ASCOM .NET Driver Templates Update (SP1)
Updated Visual Studio 2005 driver templates. These templates have the processor type set to x86 (as opposed to All CPU) as requires for COM interop. Download this file. If you used Internet Explorer, rename the file from .zip to .vsi (annoying quirk of IE!). Then double-click it, which should open the Visual Studio Template Installer. Accept the "unsigned" warning. Then for all templates, choose Overwrite Existing to replace the templates. The templates also have an improved "hint path" for the reference to ASCOM.Exceptions, which should reduce referencing problems for that assembly.
ASCOM LocalServer Template Update (1.0.2, February 2010)
The LocalServer now requests elevated privileges (if needed) when registering and unregistering on Vista and Windows 7 with User Account Control enabled. Download and simply put this zip file into My Documents\Visual Studio 200x\Templates\Project Templates\Visual C#. Do not unzip!
ASCOM Dome Control Program (maintenance update January 2010)
This release has no bug fixes or new features. It corrects an installer error. The new installer uses the ASCOM Profile component to register the program, making it compatible with ASCOM 5.5.
