Profile.WriteValue() Method

   

Write the value to the registry. Name may be an empty string for the unnamed value. Optional path to subkey for given Driver ID may be supplied.

Syntax

Profile.WriteValue(DriverID, Name, Value, [SubKey])

The method syntax has these parts:

Part Description
DriverID (String) The ASCOM DriverID for the driver (also known as the automation ProgID)
Name (String) The name of the value to be written
Value (String) The string value to be written
SubKey (String) Name of sub-key under which the value is to be written. May be a \ separated path. (optional, default = "")
Return (Nothing) Does not return a value.

Remarks

To write the unnamed (default) value, pass "" for the Name parameter. Writing the root's unnamed value is illegal as that is the driver's friendly name for the Chooser. Thus 
Profile.WriteValue("Foo.Telescope", "", Hello there")
Is illegal and will cause a trappable error to be raised. The same goes for trying to write above the level for your driver. Thus, calls with a DriverID of "" will be rejected with a trappable error.

Does not provide access to other registry data types such as binary and doubleword.
