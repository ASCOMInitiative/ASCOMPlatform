Meade Classic and Autostar I Driver

The Meade ASCOM driver is compatible with all Meade scopes except the RCX-400 and LX200GPS/R. Separate ASCOM drivers exist for the RCX-400 and LX200GPS/R scopes.

This Meade driver is actually two drivers in one. It contains both a Telescope driver as well as a Focuser driver (since the scope and focuser controls share the same serial port). The telescope driver conforms to the ASCOM Telescope V2 specifications. The focuser driver conforms to the initial (and only release at this time) of the ASCOM focuser specification.  For more information about ASCOM, see the ASCOM web site.

Which Meade driver should I use?

There are as many as four Meade type drivers that are being supported at this time:

  • Generic LX200 Type Scopes - not for use on true Meade scopes.  Many other telescopes have used control approaches based on the Meade protocols.  This driver is for them.
  • Meade RCX-400 - this is exclusively for the Meade RCX400 scopes.
  • Meade LX200 GPS/R - this is exclusively for the Autostar II Meade LX200GPS and LX200R scopes.
  • Meade Classic and Autostar I - this is the proper driver for most other Meade scopes.  (Classic LX200, old style GEMs, Autostar I).  It supports telescope and focuser functionality at the same time.

Telescope setup

The setup window (which is probably open now) allows you to adjust settings for your scope and your needs. There are two mandatory settings:

  • COM Port - This must be set to the COM port number on which your scope is connected.
  • The 16" classic checkbox - You must turn this on if you have a 16" Classic. Do not turn this on if you have a 16" LX200GPS!

Additional settings:

  • Auto Set Time (Classic only) - If set, then the driver will set the scope time to the computer time on connect.
  • Auto Unpark / Track (Classic only) - If set, the driver will unpark and set tracking to true on connect (this is the old V1 behavior, and is available for backwards compatibility).  The Autostar I always comes up unparked and tracking.
  • Beep at slew completion - The computer will beep (as well as the handbox) on completion of a slew.
  • Delay After Sync - The driver will wait for 5 seconds after a SYNC before returning control to the program. This is useful mainly with automation programs such as ACP Observtory Control Software which do things rapidly. It allows the scope some time to complete the sync and give the correct (after-sync) coordinates back to the program. Leave this off unless your software is having problems immediately after a SYNC.
  • German Equatorial (GEM) - If your mount is a GEM style mount, then check this box.  The driver will report such to higher level software which can then manage pier side details.
  • Max Slew Speed - This sets the maximum rate at which your scope will slew. This speed is internally limited by the driver to a setting of 3 for the Classic (to protect the scope in automated use). You can set a lower speed if you wish.  Putting "A" into the box is supported, and this will cause the driver to automatically set the fastest safe speed depending on your scope model.  In addition, putting an "N" into the box will completely bypass the slew speed setting in the driver, leaving your handbox setting. WARNING! Setting too high a slew speed may result in damage to your scope's nylon gears. It is recommended that you use the speed setting support in the driver! This is not supported on the Autostar I (495/497).
  • Elevation, Aperture diameter, Aperture Area, and Focal length - Some programs use this info to calculate the field of view, exposure time estimates, etc. You should fill these in. Be careful to note the units of measure!
  • Set Park Position (Classic only) - While connected, slew the scope to the position at which you want to have it park and turn tracking off. Then click this button.
  • Reset Park Position (Classic only) - See Parking (Classic section) below.

Model specifics

Classic LX200 (8", 10", and 12")

The Classic LX200 (except the 16") can have a park position remembered when used in the Northern hemisphere. Defects in the Meade firmware prevent other cases from working at this time. Firmware 3.34 or later (4.34 in the 12") are supported. Earlier firmware versions will likely not work at all. If you have an older LX200 Classic, try the Generic LX200 driver and set the encoder resolution to 0.5 arc minutes.

16" Classic

The arbitrary park position is not supported due to firmware limitations. The 16" checkbox must be turned on in the setup window or slewing will not work properly.

Autostar I #494 and #495

The Autostar I #494 an #495 must be powered-up and completely aligned (using the handbox) before connecting from the software. The Autostar I 494 an 495 cannot be unparked. After a Park command is completed, the software must be disconnected and telescope power must be cycled before normal operation can continue. Tracking state (on/off) cannot be controlled. The scope is always tracking when connected.

Autostar I #497

The Autostar I #497 must be powered-up and completely aligned (all motions and initializations complete) before connecting from the software. The Autostar I #497 can not be unparked. After a Park command is completed, the software must be disconnected and telescope power must be cycled before normal operation can continue. Tracking state (on/off) can be controlled if the most up to date firmware is installed.

Parking

The purpose of parking a scope is to allow a stationary scope to be powered up at a later time and not have to be synced or otherwise aligned. This does not apply to the Autostar I (#494, #495, and #497) which must always be aligned using the handbox before connecting the software.

For polar aligned scopes there is a default/natural Park Position.  For the Classic, the park position is at the meridian and 0-dec for polar mounted scopes and 0 alt 180 az for alt/az mounted scopes. These two are actually the same mechanical position relative to the base of the scope.

The driver is supports alternate park positions for the Classic, but the implementation is complex.  If you have any parking problems start by clicking the "Reset Park Position" button in the setup window.

LX200 Classic 8", 10", and 12" only!

The driver allows the Park Position to be changed. Once it is set, it is remembered across sessions. You don't have to set it again, unless you want to change to a new Park Position. If you don't explicitly set the Park Position, the driver will assume the default wake up position. If you power-up the scope from a different position (say from manually moving the scope axis while off), then the scope will be lost, and manual sync to the sky will be required.

  • To change the Park Position (default is 0 HA, 0 Dec):
    1. Make sure the scope coordinates are "right". In the day time you can position the scope to 0 HA, 0 Dec before turn on, or at night you can sync to any object in the sky.
    2. Position your scope at the desired Park Position. Tracking state on / off does not matter.
    3. Use one of the various [Set Park] buttons in your application software or the button in the driver setup window.  Set park has a few quirks:
      • It will be an error to set park position anywhere in the "wakeup" zone. When the LX200 Classic wakes up, it hunts for a worm start position. We need to disambiguate simple disconnect / reconnect from true cold start. The wake up zone is defined as about a degree from the 0 Hour Angle, 0 Declination point. Set park is best used when the park position is a long way from the 0, 0 point.
      • Southern hemisphere is not supported for set park.  Regular parking continues to work fine.
  • To Park:
    1. Use the [Park] command found in your software. You cannot simply move the scope to a position and turn it off. You must use a Park command. Then, do not move the scope or release the RA or Dec clutches until Unpark or connect with auto-unpark.
    2. Power down if desired.
  • To Unpark:
    1. Power up the scope if needed. Do not move it.
    2. Connect with your software. If AutoTrack is enabled, you're done.
    3. If no AutoTrack, then press Unpark.
    4. You may need to turn on tracking as well.

API notes

  • Unless listed below the Can flags will report True:
    • CanFindHome will return False
    • CanSetDeclinationRate will return False
    • CanSetGuideRates will return False
    • CanSetPark will return True for Classic scopes in the Northern hemisphere only
    • CanSetPierSide will return False
    • CanSetRightAscensionRate will return False
    • CanSetTracking will return True for Classic scopes, and the newer Autostar I #497 firmware updates
    • CanSyncAltAz will report False
    • CanUnpark will return False for the Autostar I scopes
  • CommandBlind(), CommandBool(), CommandString() support an optional "Raw" parameter. If it is not found, or set to False, then these methods will add the Meade prefix (":") and postfix ("#") characters. CommandString will also strip any trailing "#" off the return string unless Raw is set.

API notes

  • The Meade focuser is a "relative" focuser, and therefore the Position Property is not implemented.
  • The StepSize Property is not implemented since the Meade focuser does not intrinsically know its step size.
  • Temperature is unavailable
  • Temperature compensation is not supported
