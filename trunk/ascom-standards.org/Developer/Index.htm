Developer Components in the Platform

Before embarking on driver development, you should have the ASCOM Platform developer components installed on your system. When installing the Platform, be sure to enable the Developer Components checkboxes. If you didn't install them when you originally installed the Platform, you can install them by uninstalling the Platform then reinstalling, this time enabling the Developer Components. This will not back-rev any updates to the Platform you may have applied.

Role of Simulators

The ASCOM Platform includes simulators for each of the devices covered by an ASCOM standard. These simulators provide a convenient tool for application software developers to test their programs with known good drivers under controlled conditions. The simulators also serve driver developers as reference implementations of the driver standards. If there is a question about the behavior of a property or method, the behavior of the appropriate simulator serves as the reference.

Driver Developer Resources

NOTE: An update to the .NET driver templates, with a correction for 64-bit systems, is available in the Platform Updates section of the download center.

Most of the driver developer tools are contained within the ASCOM Platform 2008 as optionally installed components or as Developer Downloads in the Download Center. Driver developers are strongly encouraged to use those tools. This is particularly true for driver developers who write in C# or VB.NET. The Platform contains templates for all driver types in both C# and VB.NET.

Conformance Checker

See Conformance Testing. Once you have your driver working, it's very important to test it with the Driver Conformance Checker tool. This tool will exercise your driver and the device it handles, checking for common programming errors.
