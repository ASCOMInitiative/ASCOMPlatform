Developer Samples & Sources

The ASCOM Platform 2008 contains developer templates for Visual Studio 2005 and all types of drivers in both C#.NET and VisualBasic.NET. If you're developing a driver using VS.NET, it is strongly recommended that you use the relevant template. The Platform also contains an interface toolkit for astronomy software developers who want to use drivers from .NET.

Using the ASCOM .NET Client Toolkit from C#
A Visual Studio 2005 C# project that shows how to use the ASCOM .NET Client Toolkit. This is a console application which contains code that access each type of driver, using the chooser and pre-selecting the simulator for each. Some info is printed out to the console window. Requires the Client Toolkit 1.0.5 update.
Using the Chooser (and V5 drivers) from C++ Applications

NOTE: This works only with drivers which support the new V5 early binding (this includes all of the V5 simulators). Old drivers from ASCOM 4 do not. Some new drivers available here in the Download Center do support early binding. Look for a version number of 5.x.

Visual C++ 2005 sources and solution files for simple C++ console app that shows the easy way to use the Chooser via early binding. Also shows how to load and use a V5 driver via early binding to the ASCOM master interface. Uses ActiveX Template Library features to greatly simplify the code. The key trick is importing the Scripting Runtime library to resolve IDictionaryPtr which is used in DriverHelper.Profile. Applicable to using any of DriverHelper's classes.

POTH 5.0.4 Sources
Sources (VB6) for Plain Old Telescope Handset (POTH) 5.0.4, part of the ASCOM Platform 5 distribution. To avoid confusion and support headaches, please do not redistribute modified copies of POTH.
