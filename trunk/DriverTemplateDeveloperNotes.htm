ASCOM Template Drivers - Platform Developer Notes

The projects in this folder are used to construct the ASCOM Driver Template Projects. They are not intended to build and run directly.

Prior to developing your first driver, please familiarize yourself with the eloper.html">developer information we've provided at the ASCOM Initiative web site (internet require at the ASCOM Initiative web site (internet required).

Theory of Operation:

  1. There are three template projects:
    • C# in-process (assembly) Driver Template
    • VB in-process (assembly) Driver Template
    • C# LocalServer out-of-process (singleton) Composite Driver Template
  2. The in-process driver templates have one master template for each supported language (currently C Sharp and Visual Basic). The template code uses a number of placeholder identifiers:
     
    TEMPLATEDEVICECLASS Placeholder for the ASCOM Device Class (eg. "Focuser", "Dome").
    TEMPLATEDEVICENAME Placeholder for the ASCOM Device Name (eg. "LX200").
    TEMPLATENAMESPACE Placeholder for the project namespace.
    28D679BA-2AF1-4557-AE15-C528C5BF91E0 Placeholder Guid 1, used for the COM Interface ID, replaced with a new Guid.
    3A02C211-FA08-4747-B0BD-4B00EB159297 Placeholder Guid 2, used as the assembly Guid, replaced with a new Guid.
       
       
       

    The template source code may be edited in Visual Studio, and it should build with no errors. However, it may fail to register for COM Interop, as some of the dependencies may be unavailable during Platform development. This is not necessarily a problem. Building the template source is useful to check that no syntax errors exist in the template code.

  3. A Wizard, implemented in ASCOM.Setup.TemplateWizard, collects user input about the device being implemented and customizes the generated code during project creation. The wizard ensures that the correct interface is implemented. The wizard assembly must reside in the Global Assembly Cache on the end-user system.
  4. The template wizard deletes any files beginning with the word "Placeholder". Placeholder files can be used to create stubs and mock items that ensure the template project will compile, but which should not be present in the final generated project. Specifically, a PlaceholderInterface file is used to hold a stub interface definition which will be replaced with one of the ASCOM Interfaces in the generated project.

 

ASCOM Initiative

The ASCOM Initiative consists of a group of astronomy software developers and instrument vendors whose goals are to promote the driver/client model and scripting automation.

See the ASCOM web site for more information. Please participate in the ASCOM-Talk Yahoo Group.
