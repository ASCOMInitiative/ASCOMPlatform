Project: ASCOM Platform 6 (now a Release Candidate)

The next generation of the ASCOM Platform provides compatibility with Windows 7's enhanced security and 64-bit systems, greatly expanded support for .NET languages, some new device interfaces, and even more support for both drivers and astronomy applications. It contains new simulators, updated Visual Studio driver templates, and much more. This huge project just completed a 2 month beta test with application and driver developers, and is now available as a Release Candidate for general testing and use.

Most of you don't need to know what's going on inside the Platform. All you need to know is that it is a set of components that are used by ASCOM drivers and programs that work with ASCOM drivers. If you are a developer, or just want to know the details about the things that changed in Platform 6, here's a detailed description of ASCOM Platform 6 (May 2011).

How to upgrade to Platform 6 Release Candidate

The installer for Platform 6 has been designed to upgrade previous Platform installations (back through 4.1) without your needing to uninstall. Just download the Platform 6 Release Candidate via the link below, then run the installer. You will be able to install the final Platform 6 right over the Release Candidate.

http://download.ascom-standards.org/beta/ASCOMPlatform6RC.exe

The developer components and help have been placed into a separate installer to create a cleaner Platform install and to allow them to be updated whenever required. There is no need to install these tools if you do not develop your own ASCOM client applications or drivers.

http://download.ascom-standards.org/beta/ASCOMPlatform6DeveloperRC.exe

How to get help with problems and report bugs

If you have a problem after upgrading from an older Platform or with a new installation, first post to the ASCOM Talk Forum and ask for help. Try to supply enough info so that others can understand your problem (e.g., "ASCOM is broken" is insufficient). Have a look at How to Get Support for details on reporting problems. The Platform does not replace or modify any Windows components nor does it change settings for any program. It is strictly a support package for ASCOM drivers and programs which use ASCOM drivers.

If you are reasonably certain that you have found a bug, you can report it and file a ticket for it to be fixed. Have a look at The ASCOM Initiative Community to see how the developer community operates. To file a bug (or improvement request), use the ASCOM Jira Issue Tracker. You will need to sign yourself up for an account. Select the ASCOM Platform project and open a new issue. Don't assign anyone to it. Try to select the best fit for the type of issue. Be as detailed as possible, including a step by step repro scenario if available.

Project: ASCOM Integration with X2 and TheSky X

Software Bisque has implemented a cross-platform driver architecture called X2 that provides interfacing with astronomical devices from TheSky X Pro. The ASCOM X2 Integration project has the goal of developing generic X2 drivers that convert from X2 to ASCOM, allowing TheSky X to take advantage of all existing ASCOM drivers. The new X2 drivers will replace the old TeleAPI, FocusAPI, and DomeAPI plugins that are legacy from TheSky 6. The X2 interfaces provide a richer set of capabilities, allowing TheSky X to take advantage of more features of devices via their ASCOM drivers. In development are generic X2 to ASCOM drivers for Mounts, Focusers, Cameras, FilterWheels, Rotators, and Domes. Some of the X2 interfaces themselves are evolving, so we're tracking these developments. Look for the first set of these X2 to ASCOM drivers early Summer 2011.

Note that ASCOM is Windows-only, so these X2 to ASCOM drivers will not work with TheSky X Pro running on Mac OS X.
