ASCOM Camera Gain, Offset and SubExposure - RFC

Within the astronomy community there are numerous motorised telescope covers, flat calibration panels and "combination devices" that provide both "cover" and "calibrator" capabilities, but ASCOM has no dedicated interface for these devices. Some manufacturers have made use of the Switch interface, but there is no informal naming or usage standard in place that provides a consistent interface upon which applications can rely.

This document describes the new ASCOM interface: "ICoverCalibrator", which will support telescope covers, calibration panels and combination devices and which is the outcome of a public RFC process conducted at the end of 2019.

Approach Rationale

Telescope covers and calibration panels can be modelled straightforwardly because they focus on just one specific capability, however, the combination products pose a challenge because they deliver two distinct capabilities from one device. Consideration was given to creating two separate interfaces, but this approach was rejected because a single combination style device would end up with two sets of common properties such as "Connection" and this would inevitably lead to confusion for users, client authors and device manufacturers.

To address this situation, a single interface will be delivered that comprises two capability groups, one for each of the distinct functional capabilities. This ensures that all devices continue to have a single set of common properties and that the interface is sufficiently flexible to accommodate both single capability and multi-capability devices.

The interface has been designed to support Asynchronous Operation and has status properties through which drivers can reveal the capabilities that their devices support.
